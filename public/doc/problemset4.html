Web Application Engineering - Group 4

Software Structure

Content
Potential content available from the website:
  • Student and Sites descriptions
  • Articles and Reviews on different internship sites including comments and feedback
  • Evaluation on different internship sites including rating
  • Evaluation on students given by sites students have worked for

Technology
The website will use the Apache Web Server with Mongrel cluster, Ruby on Rails as the scripting languages and Postgresql as the database back-end. The website use SSL for sending user sensitive information.
The basic information processing requirements:
  • User databases, stores user-related information : identification code, name, email address, phone numbers, addresses, and others.
  • Content database, stores the core content of the website : articles, reviews, rating, comments, feedback, access controls, and others.
  • User/Content map, stores information about contents related to users, sends notification when someone comment or respond to the articles and reviews, sends evaluation notification to different sites, moderate the contents, administrative/modification rights, and others.

Questions:
Do you store users' passwords in the database encrypted or non-encrypted?
Yes. We use SHA1 HexDigest (as defined in Lab 4-UserAuth). We did not implement RESTFul or AuthLogic for Authentication. Our project requirement suggest Administrator have full privilege over the registration of all user viz. Student, Faculty and Site. While sending password during Login and Changing password, SSL is used to prevent eavesdropping.

What are the advantages and disadvantages of encryption?
Advantages:
  • Increase security.
  • Prevent others to know the password even when s/he have access to database.
  • Protect important data when transmitted over the Internet.
Disadvantages:
  • Encryption takes time. The more complex the encryption the more processing it require.
  • Encrypted/Hashed password are hard to recover when lost. Some different methodology is need to implement when user forget password.
  • Difficult to Database Administrator. When content are encrypted before storing into database, Database administrator have some trouble understanding the content of the database.
What columns in your tables will enable your system to handle the query "Find me users who live within 50 kilometers of User #37"?
We have normalized the User database. We need to do table join to query.

Questions:
Can someone sniffing packets learn your user's password?
No. We have implemented SSL for help prevent eavesdropping while sending user sensitive information.

Gain access to the site under your user's credentials?
Website uses Username and Password for Authentication. Only verified users are allowed to access the restricted areas.

What happens to a user who forgets his or her password?
When user forget the password, s/he request for change in password verifying his/her username. The system will send the validation code to his/her email address which been registered to that username. After receiving the validation code, the user can now change his/her password by verifying the validation code. Each validation code have validity period .
